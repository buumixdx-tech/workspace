# 业务概述

## 业务背景

**公司运营饮料机业务**，业务模式为：

- **设备**：公司采购饮料机设备，投放到客户运营点位部署运营
- **物料**：公司采购饮料物料（粉浆等），客户向我们采购饮料物料
- **盈利模式**：通过物料差价盈利（公司低价采购物料，高价销售给客户）

公司既是设备/物料的采购方，也是面向客户的供应方，同时管理设备资产、客户点位、物流配送和资金收付。

---

## 工作流程

```
信息录入（主数据）→ 业务管理（签约流程 6 阶段 + 附加政策 + 供应链）
                                    ↓
业务开展（6 类 VC 操作）
├── 设备采购     → 客户押金 + 向供应商付款 + 物流入库（SN 登记）
├── 物料供应     → 扣减库存 + 客户预收/履约收款
├── 物料采购     → 补自有库存 + 向供应商付款
├── 设备采购(库存) → 补自有设备库存 + 向供应商付款
├── 库存拨付     → 仓库间调拨设备（无资金流）
└── 退货         → 逆向物流 + 退款/供应商退货
                                    ↓
具体运营
├── 物流管理     → 入库确认触发三联动（库存 + VC 状态机 + 财务记账）
├── 资金流管理   → 收付款推进 VC.cash_status，复式记账生成凭证
└── 财务信息     → 自身运营账 + 客户/供应商分户账
```

## VC 类型（6 类）

| VC 类型 | 含义 | 资金方向 |
|---------|------|---------|
| `设备采购` | 设备采购（关联业务） | Customer→我们→Supplier |
| `设备采购(库存)` | 设备采购（不关联业务） | 我们→Supplier |
| `库存拨付` | 仓库间调拨 | 无 |
| `物料采购` | 物料采购 | 我们→Supplier |
| `物料供应` | 向客户供应物料 | Customer→我们 |
| `退货` | 退货执行单 | 反向 |

## 核心驱动机制

- **VC 三状态机**（`status` / `subject_status` / `cash_status`）：独立推进、联动完成
- **三联动核心节点** `confirm_inbound_action`：库存更新 + VC 状态机推进 + 财务记账同时触发
- **所有 Action 支持快照回滚**：操作事务记录 `snapshot_before`/`snapshot_after`，支持幂等回滚/撤销回滚
