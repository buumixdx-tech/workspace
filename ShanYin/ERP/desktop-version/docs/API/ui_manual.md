# 闪饮 ERP 系统 UI 操作手册 (全家桶)

本手册旨在提供闪饮 ERP 系统中所有用户交互界面的详尽说明，涵盖从基础数据录入到复杂业务执行的全流程操作。

---

## 1. 系统导航与全局交互

### 1.1 侧边栏导航
- **控制中心**: 点击右上角或左侧边栏图标切换主要功能模块。
    - **运行看板**: 业务统计、合同追踪、待办事项提醒。
    - **基础录入**: 归属主体、点位、客户、供应商、产品/协议管理。
    - **业务管理**: 供应链导入、业务项目（包含点位关系）开启。
    - **运营中心**: 虚拟合同执行（采购、拨备、退货、拨付）。
    - **物流管理**: 物流单生成、状态追踪、入库办理。
    - **资金管理**: 流水录入、应收/应付概览、冲抵池监控。
    - **财务管理**: 银行账户配置。

### 1.2 通用交互模式
- **Data Editor (网格编辑)**:
    - **手动录入**: 在网格内直接输入数据。
    - **动态增减**: 使用表格底部的 `+` (Add row) 按钮。
    - **下拉选择**: 某些列提供 Selectbox 选项（如品类、仓库、点位）。
- **确认对话框 (Streamlit Dialog)**:
    - 关键操作均有二次确认弹窗，展示核心明细，需点击“确认”方可执行。
- **状态看板 (St.Metric/HTML Cards)**:
    - 顶部或卡片内展示关键财务状态（已付、待付、冲抵额）。

## 2. 基础数据录入 (Entry)

### 2.1 归属主体管理
- **查看**: 列表展示所有内部核算主体。
- **操作**: 
    - **添加**: 输入名称、类型、地址、收货地址、联系方式。
    - **编辑**: 点击列表项进行信息修改。

### 2.2 客户与供应商管理
- **操作**: 同归属主体，支持设置独立收货地址。

### 2.3 供应链 (Supply Chain) 导入
- **流程**: 
    1. 选择供应商。
    2. 设置“价格协议” (Data Editor): 录入 SKU 名称、协议价、起订量。
    3. 设置“结算协议”: 预录预付比例、账期时限、起算规则（如物流签收后）。
    4. 设置“时间规则”: 预设 SLA 约束。

### 2.4 产品与点位管理
- **产品 (SKU)**: 录入产品规格、单位、默认分类。
- **点位**: 录入具体仓位、客户服务点位置、经纬度信息。

---

## 3. 业务项目管理 (Project Inclusion)

### 3.1 业务引入流
- **第一步 (基本信息)**: 选择归属主体、客户、业务负责人。
- **第二步 (点位配置)**: 从点位池勾选该业务涉及的所有物理点位。
- **第三步 (规则预设)**: 为本项目设置全局时间规则模板。
- **执行**: 点击“确定并开启业务项目”。

### 3.2 业务看板与纠偏
- **查看详情**: 点击业务条目查看各模块进度。
- **数据纠偏**: 在“数据纠偏” Tab 下，手动强制修正项目状态或基础参数。

---

## 4. 虚拟合同运营 (Operations)

### 4.1 采购执行
- **设备/库存采购**: 
    1. 选择供应链。
    2. 挑选 SKU。
    3. 分配入库点位（仓库）。
    4. “锁定采购单”后进入确认弹窗。
- **物料采购**: 
    - 类似于设备采购，但侧重于耗材。

### 4.2 资产分配 (拨备/拨付)
- **拨备 (从项目)**: 针对具体业务需求，挑选供应商 SKU 分配至客户点位。
- **库存拨付**: 将“自有仓”中已有的库存资产分拨给指定客户点位。

### 4.3 退货操作 (Return)
- **流程**:
    1. 选择关联虚拟合同。
    2. 勾选需要退货的物品。
    3. 选择退货方向（退回我方仓库/退回供应商/点位间调拨）。
    4. 系统自动计算应退款项及押金释放。

---

## 5. 物流与仓储 (Logistics)

### 5.1 物流计划下达
- **初始化建议**: 按钮触发，系统自动将合同物品按点位/仓库进行快递单分组。
- **快递单调整**: 修改单号（临时号改为实单号）、复核地址。
- **生成**: 点击“确认生成发货单”。

### 5.2 物流追踪
- **推进状态**: 支持“一键批量推进”（待发货 -> 运送中 -> 已签收）。
- **手动修正**: 单个快递单点击“修改”修正路由信息。

### 5.3 办理入库 (关键节点)
- **确认收货**: 当快递单处于“已签收”时，点击“办理入库”。
- **SN 录入**: 针对设备及资产，必须在文本框中粘贴 SN 码列表。
- **同步**: 点击“准予结单并同步系统”，系统将正式增加库存并将合同状态转入“结案/待财务结算”。

---

## 6. 资金流管理 (Finance)

### 6.1 流水录入
- **选择合同**: 仅显示“执行中”的合同。
- **核算看板**: 悬浮窗展示当前合同：应收/付总额、已收/付金额、待收/付余额、冲抵池情况。
- **账户建议**: 系统智能建议收付款主体，用户选择银行账户。
- **保存**: “确认录入”后流水即刻生效并累加至合同进度。

### 6.2 财务概览
- **看板**: 全局展示最近 20 条大额资金变动。
- **检索**: 支持按款项性质（预付、尾款、押金、罚金）过滤。

---

## 7. 规则与时间约束管理

### 7.1 通用规则管理器
- **入口**: 出现在 SC 导入、业务引入、虚拟合同、物流任务的“规则管理” Tab。
- **触发器设置**:
    - 选择“触发事件”（如：合同签约、付款完成、物流签收）。
    - 选择“目标事件”（如：货物送达、余款支付、项目上线）。
- **约束配置**:
    - 设置偏移时间（如：签收后 **7个工作日** 内）。
    - 设置责任方（我方/供应商/客户）。
- **覆盖逻辑**: 本级设置的规则将优先于从父级继承的模板规则。

---

> [!IMPORTANT]
> **操作建议**：
> 1. 所有包含表格（Data Editor）的页面，在点击“提交/保存”前，请务必确认表格内最后一行数据已失去焦点（点击表格外空白处），否则可能导致最后一行数据未被采集。
> 2. 办理入库时录入的 SN 码应保持一行一个或以逗号分隔，以确保资产建档逻辑准确。
