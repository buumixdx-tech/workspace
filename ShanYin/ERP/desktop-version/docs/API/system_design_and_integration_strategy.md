# 闪饮业务管理系统 (ShanYin ERP-v4) 系统设计与未来集成策略

## 第一部分：系统现状设计与功能描述

### 1. 系统架构设计
*   **前端框架**: 基于 **Streamlit** 的响应式 Web 界面，整合了 `streamlit-antd-components` (SAC) 以提供高级 UI 交互。
*   **后端语言**: Python 3.x。
*   **数据库**: **SQLite** (分为生产库 `business_system.db` 和测试库 `test.db`)。
*   **持久层**: 使用 **SQLAlchemy ORM** 进行对象关系映射，支持复杂的关联查询和事务。
*   **设计模式**:
    *   **VC 中心化**: 所有业务动作（采购、供应、退货）均通过虚拟合同（Virtual Contract）承载。
    *   **状态机 (State Machine)**: 驱动业务状态、标的状态和资金状态的自动变迁。
    *   **事件驱动 (Event-Driven)**: 核心操作会触发 `SystemEvent`，用于日志审计、AI 消费及跨模块联动。

### 2. 核心功能模块

#### 2.1 商务中心 (Business & Customer)
*   **渠道客户管理**: 记录 B 端客户信息、合作详情。
*   **业务生命周期跟踪**: 跟踪业务从“前期接洽”到“评估”、“落地”及“正常开展”的全过程。
*   **点位/仓库管理**: 精细化记录客户运营点位、自有仓、供应商仓及转运仓的地理信息与收货联系。

#### 2.2 供应链与 SKU 体系 (Supply Chain)
*   **主数据维护**: 统一管理设备类和物料类 SKU，支持参数 JSON 存储。
*   **供应商协议**: 维护与供应商的长期合作协议，包括阶梯价、浮动价及支付账期配置。
*   **物料采购/供应**: 基于供应链协议快速生成采购计划。

#### 2.3 虚拟合同执行 (Virtual Contract - 核心驱动)
该模块将复杂的长期合同拆解为可操作的执行单元，独立跟踪三个维度的状态：
*   **业务总状态**: 执行、完成或终止。
*   **标的状态 (Subject Status)**: 物流层面的发货、签收、完成。
*   **资金状态 (Cash Status)**: 财务层面的预付、结清。

#### 2.4 库存与物控 (Inventory Management)
*   **设备库存 (SN 级)**: 针对每一台机器进行序列号 (SN) 管理，实时跟踪其在库、运营或维保状态。
*   **物料库存 (统计级)**: 按仓库维度分布管理物料数量，支持加权平均价计算和低库存预警。
*   **库存拨付**: 支持自有库存与运营点位间的灵活调拨。

#### 2.5 物流与执行 (Logistics & Express)
*   **物流计划**: 与虚拟合同绑定，支持一单多邮，精细化记录快递明细。
*   **实时签收反馈**: 快递签收自动联动虚拟合同的标的状态变迁。

#### 2.6 财务与资金流 (Finance & Accounting)
实现了标准会计准则的复式记账体系：
*   **复式记账**: 所有的资金流（收付款）和标的流（进出库）自动生成借贷凭证。
*   **会计科目管理**: 预设收账、应付、存货、实收资本等标准科目。
*   **现金流量表**: 自动统计经营性、投资性、融资性三大现金流。

#### 2.7 时间规则与合规引擎 (Time Rule Engine)
*   **自动化监控**: 基于合同约定生成监控规则（如：签收后 30 天内必须付款）。
*   **四级预警**: 提供绿、黄、橙、红四级告警，并在系统侧边栏实时滚动提醒告警任务。

## 第二部分：面向 AI Agent 的 API 化集成建议

### 1. 架构愿景：从“UI 驱动”转向“服务驱动 (Headless)”
*   **建议**: 引入 **FastAPI** 作为并行的服务入口。
*   **做法**: 保持 `logic/` 文件夹的代码纯净（无 `st.xxx` 字样）。Streamlit 页面和 FastAPI 路由共享同一套逻辑服务层（Service Layer）。

### 2. 打造“语义化” AI-Ready API
AI Agent（如 OpenClaw）需要高级别的工具描述，而非原始的数据库操作：
*   **工具化定义**: 为 Agent 提供 `action/procure-material` 等语义明确的高级接口。
*   **自描述 Schema**: 利用 Pydantic 的 `Field(description="...")` 属性，为每个字段添加详尽的业务描述，直接供 AI 模型读取。

### 3. 主动交互：事件流与 Webhook
*   **建议**: 将 `SystemEvent` 表转化为外部可订阅的 **Webhook**。
*   **场景**: 当出现“红色告警”时，系统主动推送事件，AI Agent 接收到通知后自动执行预设动作（如发起催款或补货）。

### 4. 智能化辅助：上下文检索 API
*   **建议**: 提供一个 `GET /query/context` 接口。
*   **功能**: 为 Agent 提供决策支持，返回关联的 VC 状态、资金流水和异常历史，帮助 AI 理解“为什么要这么做”。

### 5. OpenClaw 特别优化建议
*   **全景感知接口**: 建议设计一个 `GET /system/status` 接口，一键返回当前所有高优先级（红色告警）任务和待处理财务单据，让 Agent 能够像人类主管一样快速梳理今日工作重点。
