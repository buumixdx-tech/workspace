## 一、信息录入

主页面含 5 个标签页，对应五大基础信息表：

| Tab 名称 | 对应表 | 功能 |
|---------|--------|------|
| 渠道客户 | `channel_customers` | 新建/维护客户信息 |
| 点位 | `locations` | 新建/维护部署点位 |
| 供应商 | `suppliers` | 新建/维护设备/物料供应商 |
| SKU | `skus` | 新建/维护商品/设备规格 |
| 外部合作方 | `external_partners` | 新建/维护第三方合作方 |

> 🔹 所有表支持 CRUD 操作，数据供后续业务模块引用。

## 二、业务管理

### 2.1 客户导入流程

客户导入是签约前的推进阶段，共 **4 个状态阶段**，形成线性流程，支持「推进」与「取消」两条路径。

```mermaid
flowchart TD
    A[前期接洽] -->|推进| B[业务评估]
    B -->|推进| C[客户反馈]
    C -->|推进| D[业务落地]
    D -->|推进| E[业务开展 ★]
    
    A -->|取消| Z[业务终止]
    B -->|取消| Z
    C -->|取消| Z
    D -->|取消| Z
    
    classDef stage fill:#e6f7ff,stroke:#1890ff;
    classDef end fill:#f6ffed,stroke:#52c41a;
    classDef cancel fill:#fff1f0,stroke:#f5222d;
    
    class A,B,C,D stage;
    class E end;
    class Z cancel;
```

#### 状态说明

| 状态 | 操作 | 行为 |
|------|------|------|
| 前期接洽 | 推进 | 录入接洽说明 → 进入`业务评估` |
|          | 取消 | 录入取消原因 → 进入`业务终止`（从列表隐藏） |
| 业务评估 | 同上 | 录入评估结果 → 进入`客户反馈` |
| 客户反馈 | 同上 | 录入反馈结果 → 进入`业务落地` |
| 业务落地 | 推进 | 创建合作合同（PDF+JSON）+ 外部合作合同 → 创建时间规则 → 进入`业务开展` |
|          | 取消 | 同上 → `业务终止` |

> 🔹 所有操作说明均带时间戳写入业务 JSON 字段  
> 🔹 合同创建后，系统自动生成：  
>   ✓ 合同表条目  
>   ✓ 文件夹（存 PDF/JSON）  
>   ✓ 时间规则表条目

### 2.2 供应链管理

#### 2.2.1 建立供应链关系

```mermaid
flowchart LR
    A[上传供货合同 PDF/JSON] --> B{合同类型？}
    B -->|设备| C[创建设备供应链]
    B -->|物料| D[创建物料供应链]
    C & D --> E[创建外部合作合同*]
    E --> F[输入时间规则]
    F --> G[创建合同 + 供应链 + 时间规则条目]
```

#### 2.2.2 供应链更新与终止

```mermaid
flowchart TD
    S[生效中供应链] -->|更新| U[上传新合同]
    U --> U1[校验：<br/>① 供应商/类型一致<br/>② 签订日期更新<br/>③ 无超越条款]
    U1 -->|通过| U2[创建新合同+时间规则<br/>旧合同状态=“终止”]
    
    S -->|终止| T[录入终止原因]
    T --> T1[关联合同状态=“终止”]
```

> ⚠️ 更新时硬性校验 ①②，③ 仅预警。

## 三、业务开展

客户签约完成后，时间引擎自动将“生效中”业务推入此模块。

### 主页面三标签：

| Tab | 内容 |
|-----|------|
| 业务列表 | 所有“生效中”业务，可发起 **设备采购**/**物料供应** |
| 物料采购 | 独立库存补货操作，查看 SKU 库存总量 |
| 退货操作 | 对“合同标=完成”的虚拟合同发起退货 |

---

### 3.1 设备采购

#### 操作约束：
- ✅ 每次采购 **1 种 SKU**
- ✅ 仅可从 **1 条供应链**下单
- ✅ 部署点位可多选

#### 流程：

```mermaid
flowchart TB
    P[选择业务 → 设备采购] --> F[填写表单：<br/>SKU, 数量, 点位, 供应链]
    F --> C[押金信息<br/>（取自合同，可调+原因）]
    C --> T[输入时间规则]
    T --> V[创建虚拟合同]
    
    V --> V1[合同状态=“执行”]
    V --> V2[合同标=“执行”]
    V --> V3[资金流状态：<br/>有预付→“执行”，无→“已预付”]
    V --> V4[创建时间规则]
```

> 🔹 虚拟合同：`供应商 → 我们 → 客户点位` 的直发链路  
> 🔹 后续由物流/资金流驱动状态机变更

---

### 3.2 物料供应

#### 流程：

```mermaid
flowchart TB
    PS[选择业务 → 物料供应] --> F1[选多个SKU<br/>（限业务关联SKU）]
    F1 --> F2[填数量、点位]
    F2 --> Ck[库存检查]
    Ck -->|不足| Err[提示 → 返回]
    Ck -->|充足| T[输入时间规则]
    T --> V[创建虚拟合同]
    
    V --> V1[合同状态=“执行”]
    V --> V2[合同标=“执行”]
    V --> V3[资金流状态同设备采购]
```

---

### 3.3 物料采购（补库存）

```mermaid
flowchart LR
    MP[物料采购 Tab] --> F[选供应链]
    F --> Items[填多个SKU/数量/价格/发货]
    Items --> T[时间规则]
    T --> VC[创建虚拟合同]
    
    VC --> State[状态：“执行”]
    VC --> Inv[后续触发库存增加]
```

> 🔹 可多次发起，支持多供应链分批采购。

---

### 3.4 退货操作

#### 触发条件：
- 仅对 **合同标状态 = “完成”** 的虚拟合同

```mermaid
flowchart TB
    R[退货 Tab] --> List[列出可退合同]
    List --> Btn[点击“退货”]
    Btn --> Form[填退货信息：<br/>SKU/数量/点位/退款/物流费/说明]
    Form --> Sub[提交]
    
    Sub --> NewVC[创建退货虚拟合同]
    NewVC --> N1[合同状态=“执行”]
    NewVC --> N2[资金流状态=“已预付”]
    NewVC --> N3[关联原合同ID]
```

## 四、具体运营

### 4.1 物流管理

#### 物流状态机概览：

```mermaid
stateDiagram-v2
    [*] --> 待发货
    待发货 --> 在途 ： 填快递单号
    在途 --> 签收 ： 推进
    签收 --> 已签收 ： 所有快递签收
    已签收 --> 完成 ： 入库 + 上传凭证
    完成 --> [*]
```

#### 关键校验：
- ✅ 单 SKU 发货量 ≤ 虚拟合同约定量  
- ✅ 总发货量 ≤ 合同总量  
- ✅ 设备采购入库需上传 **设备编号 JSON**

> 🔹 每次快递单推进 → 触发物流单状态机 → 触发虚拟合同状态机

---

### 4.2 资金流管理

#### 新建资金流流程：

```mermaid
flowchart TB
    NF[新建资金流] --> Sel[选虚拟合同]
    Sel --> Pay[填付款方/账户、收款方/账户]
    Pay --> Type[选类型：<br/>押金/履约/罚金/退还押金]
    Type --> Amount[金额+时间+说明]
    Amount --> Proof[上传凭证]
    Proof --> Save[创建资金流条目]
    
    Save --> Trig1[触发虚拟合同状态机]
    Save --> Trig2[触发财务模块]
    Save --> Trig3[押金类 → 押金模块]
```

## 六、内部模块

### 6.1 时间引擎

```mermaid
flowchart LR
    Engine[时间引擎] --> Contracts[监控合同：<br/>生效/到期]
    Engine --> VContracts[监控虚拟合同：<br/>物流/打款节点]
    Engine --> Forecast[客户物料预测]
    
    Contracts --> Alert1[到期预警]
    VContracts --> Alert2[超时提醒]
    Forecast --> Alert3[补货建议]
```

> 🔹 规则优先级：具体时间规则 > 内置规则  
> 🔹 系统启动/每日定时触发扫描

---

### 6.2 状态自动机

#### （1）物流单状态机

```mermaid
flowchart TD
    In[快递单状态变更] --> Check[检查所有快递单状态]
    Check -->|全“在途”| Ship[物流单→“已发货”]
    Check -->|全“签收”| Signed[物流单→“已签收”]
    Signed -->|入库| Done[物流单→“完成”]
    
    Ship & Signed & Done --> Trig[触发虚拟合同状态机]
```

#### （2）虚拟合同状态机（简化）

```mermaid
flowchart LR
    VC[虚拟合同] -->|物流签收| VC1[标状态=“签收”]
    VC1 -->|物流完成| VC2[标状态=“完成”]
    VC -->|资金足额| VC3[资金流=“完成”]
    VC2 & VC3 --> VC4[合同状态=“完成”]
```

---

### 6.3 财务模块（逻辑摘要）

| 入参 | 行为 |
|------|------|
| **物流单（完成）** | 按合同类型生成：<br/>✓ 存货增减<br/>✓ 应收/应付款<br/>✓ 成本/收入 |
| **资金流** | 按类型生成：<br/>✓ 银行存款变动<br/>✓ 现金流分类<br/>✓ 营业外收支 |

> 📌 示例：  
> - `设备采购+物流完成` → 存货↑ + 应付↑（=合同额−预付）  
> - `物料供应+资金履约` → 应收↓ + 银行存款↑

### 6.5 押金管理模块

#### 双触发路径：

```mermaid
flowchart TB
    A[物流完成] -->|设备采购| Calc1[按设备数均摊押金]
    B[押金资金流] --> Check[时间戳校验]
    Check -->|有效| Calc2[调整总押金 + 均摊]
    Calc1 & Calc2 --> Update[更新库存条目押金字段]
```

> 🔹 押金 = 虚拟合同押金总额 ÷ 关联设备数
